# 公益性项目完整场景与 Prompt 讲解

[返回总目录](./README.md)

## 场景设定

以一个典型的公益型双创项目为例：团队计划做一个面向乡村留守儿童的 AI 阅读陪伴与心理关怀平台，希望通过低成本、可持续的数字化服务改善阅读资源不足和陪伴缺失问题。

这类项目与商业型项目不同，它的首要目标不一定是利润最大化，而是社会价值实现。但它同样不能停留在“有意义”这一层面，还必须回答几个更现实的问题：受益对象是谁、问题是否足够具体、项目成效如何衡量、项目如何持续运行。

因此，在公益场景下，智能体会把项目放进另一条更适合社会创新项目的推理链中理解：

**受益人群 -> 社会问题 -> 干预方案 -> 影响指标 -> 资源来源 -> 合作网络 -> 持续运营机制**

## 系统如何理解这个场景

公益项目并不是脱离创业逻辑的例外，而是创业逻辑的另一种目标函数。商业项目关注“市场闭环”，公益项目则更强调“影响力闭环”和“可持续闭环”。

因此，系统在这一场景下会重点判断：

1. 受益群体是否明确，而不是只讲“帮助更多人”。
2. 社会问题是否被具体定义，而不是只有价值口号。
3. 干预方案是否真正作用于问题，而不是只做表层陪伴。
4. 项目效果是否可以衡量，例如覆盖人数、频次、行为变化、反馈结果。
5. 资源支持与合作网络是否存在，例如学校、社区、公益组织、企业 CSR 等。
6. 项目是否具有持续运行机制，而不是只依赖一次性补贴或短期热情。
7. 是否存在隐私、伦理、治理等敏感风险。

换句话说，公益项目并不是不看“能不能跑起来”，而是把“持续运行”和“社会成效”放到了比盈利更核心的位置。

## 扩展设计说明

虽然系统中未必显式设置了一个单独的“公益项目模式”，但从文档层面可以把它总结为一套合理的扩展设计。

### 1. 双闭环判断

公益项目需要同时满足两种闭环：

- **社会价值闭环**：问题真实存在、干预方案有效、结果可衡量。
- **组织可持续闭环**：资金、合作方、志愿者、社区与平台能力能够支撑项目长期运行。

这意味着公益项目不能只讲初心，还必须讲机制。

### 2. 影响力证据优先

对于公益项目，系统中的“证据”不再局限于营收和付费数据，还包括：

- 受益对象的真实反馈
- 前后对比数据
- 服务覆盖人数与服务频次
- 行为改变或结果改善指标
- 合作机构反馈

这类证据决定项目是否真的带来了社会改变，而不只是提出了一个有温度的故事。

### 3. 用可持续性替代单一盈利性

商业项目通常优先追问“怎么赚钱”，公益项目则更适合优先追问“怎么持续”。因此，Prompt 设计里会更加关注：

- 谁在为项目提供资源
- 项目依赖哪些合作主体
- 如果短期补贴结束，项目是否还能继续运行
- 是否存在校地合作、公益基金、企业 CSR 或轻量收费等复合支持方式

## Prompt 设计思路

公益项目的 Prompt 设计重点，是把模型从“商业评估单一路径”切换到“社会影响 + 持续运营”的双路径判断上。

### 1. 身份定位

可以把模型设定为“公益创新项目教练”“社会影响力评估顾问”或“公益项目路演辅导者”，让模型天然关注社会问题定义、影响评估和可持续机制。

### 2. 分析维度约束

要求模型重点检查：

- 目标受益人群是否清晰
- 社会问题是否具体而非口号化
- 干预机制是否清楚
- 影响结果如何衡量
- 合作方与资源网络是否稳定
- 持续运行机制是否存在
- 合规、伦理和数据安全是否充分

### 3. 输出结构约束

除了常规的瓶颈、追问、下一步任务外，还可以要求模型额外输出：

- 核心社会价值指标
- 当前最需要补的影响力证据
- 最脆弱的持续运行风险点

这样可以避免公益项目被简单处理成“有意义，所以值得做”，而是把它真正分析为“如何做成、如何持续、如何证明有效”。

## 典型 Prompt 示例

```text
你是一名公益创新创业项目教练，请从“社会影响力”和“可持续运行机制”两个维度分析该项目。
不要只评价项目是否有意义，而要重点判断：
1. 目标受益人群是否清晰；
2. 社会问题是否被具体定义；
3. 干预方案是否真正作用于问题；
4. 影响结果能否被衡量；
5. 项目是否有可持续的资源与合作机制；
6. 是否存在伦理、隐私或治理风险。

请输出：
- 当前最关键的瓶颈
- 3个启发式追问
- 1个下一步唯一任务
- 影响力验证的验收标准
```

## 这种 Prompt 为什么有效

因为公益项目最容易出现两类问题：第一类是“价值叙事很强，但证据很弱”；第二类是“初心很强，但机制很弱”。前者会导致项目只有情怀表达，没有真实影响证明；后者会导致项目短期热闹、长期失速。

因此，这类 Prompt 的真正作用，是完成两次压实：

- 把“情怀表达”压实为“影响力证据”。
- 把“公益愿景”压实为“持续运行机制”。

## 可写入最终文档的总结

在公益性项目场景下，智能体强调项目是否形成了“社会问题 - 干预方案 - 影响评估 - 持续运营”的影响力闭环。系统不会只把公益项目理解为“有意义的想法”，而是通过结构化 Prompt 追问项目是否真正识别了受益对象、是否设计了有效干预、是否具备可衡量结果、是否拥有持续推进所需的资源与合作网络。由此，模型在公益场景中的价值，不只是帮助表达公益愿景，更是帮助团队把公益愿景落到可验证、可持续、可执行的方案上。
